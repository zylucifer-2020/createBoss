# Boss Judgment 分析 Prompt

## 任务

根据老板的聊天记录、会议纪要、邮件、批注和项目文档，提取他判断项目的标准。

目标不是总结“他很严格”，而是总结：

- 他怎么判断项目值不值得做
- 他怎么判断方案是否过关
- 他怎么判断风险是否可接受
- 他怎么判断一个人是否靠谱

## 提取维度

### 1. 项目价值判断

- 他最先看什么：impact、收入、效率、风险、战略意义、老板要求
- 什么样的项目他会支持
- 什么样的项目他会否掉

### 2. 方案评审标准

- 他开会时最常问的问题
- 他最在意的信息是否齐全
- 他喜欢什么结构的方案
- 哪些表述会让他不耐烦

### 3. 执行与推进

- 他如何看待 owner 感
- 他如何看待延期
- 他如何看待风险暴露
- 他如何看待跨团队协同

### 4. 资源与优先级

- 什么情况下他愿意给资源
- 什么情况下他会说“先别做”
- 他如何做取舍

### 5. 对人的判断

- 他怎么判断一个下属是否靠谱
- 什么行为会加分
- 什么行为会减分

## 输出要求

- 语言：中文
- 所有结论尽量写成可执行规则
- 有原话依据时直接标注
- 信息不足时写：`（原材料不足，建议追加会议纪要或项目评审记录）`
